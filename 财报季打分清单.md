> 用途: 财报出来时逐票对照,验证框架是否有效。入场价快照于 2026-07-08。
> 每票发财报后: ①跑 `scan.py track --action grade` 拿量化回报 ②对照本表的确认/否决条件做定性打分。

## 打分规则（每票财报后判定）
- **确认✅**: 满足"确认条件" → 按卡加仓至目标
- **证伪❌**: 触发"一票否决线" → 清仓,并在 lessons.md 记录哪个筛选器给了假信号
- **中性➖**: 都没触发 → 维持,继续观察
- 汇总: 确认率、以及"否决股 vs 确认股的后续回报"(检验过滤器松紧)

## 记分卡机制（量化回报,与本表定性打分互补）
```bash
cd scanner
../.venv/bin/python scan.py track --action grade   # 拉最新价,算各票自入场回报
../.venv/bin/python scan.py track --action score   # 汇总胜率/均值 → scorecard.md
../.venv/bin/python scan.py audit                  # 体检+缺口自问(含③否决股vs买入股回报)
```
**关键检验(audit③)**: 若被否决的股(SYM/VLO/STNG/RGLD/CORZ...)后续回报 > 买入股均值 → 过滤器过严;反之过松。这才是框架"自证或自否"的终极测试。
