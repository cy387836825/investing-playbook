# 自进化机制（Self-Evolving Mechanism）

> 目标: 让这套系统不止是静态流水线,而是会**纠错、提效、发现遗漏**的活系统。
> 建立于 2026-07-08。三个回路 + 一套升级纪律。

## 为什么需要它
整个研究过程中,大量流水线缺陷是**手动发现、散落各处**的（S1的TTM滞后、S2b并购注水、商品股滞后峰值、被收购标的、生物药里程碑注水……）。若不系统化,同样的错会重复犯,好的改进也不会沉淀。自进化机制把"发现→修正→固化→验证"变成闭环。

---

## 回路1: 纠错（Error-Correction）→ `lessons.md`
**机制**: 每次深挖/否决中发现的陷阱,登记进教训台账,标注 `[已自动化]/[手动]/[待自动化]`。
**升级纪律（核心）**: **同一陷阱手动命中≥2次 → 必须升级为 `[待自动化]` → 下次迭代写进 scan.py**。
不让人(或AI)反复做机器能做的检查。

## 回路2: 提效（Efficiency）→ 自动化进 `scan.py`
把回路1里"毕业"的教训写成代码,report自动输出,不再手动核。**本次已自动化**:
- L1 周期阶段分层（stage字段,修TTM滞后）
- L5 跳变检测（`_lumpy_flag`: ni_series符号翻转/CV>1.5 → 标"一次性/里程碑/并购嫌疑"）
- L3 非内生桶（金融/材料/能源/REIT自动标"营收概念存疑"）
- L9 分诊分桶（按加速度排序而非新闻显著性,反选择偏差）
- L10 相关性聚类（report输出每板块命中数,≥5标"🔴组敞口上限"）

**待自动化队列**（audit会提醒,见lessons.md）: L2价格位置入报告、L11修正动量入报告、L8利好不涨。

## 回路3: 发现遗漏（Gap-Discovery）→ `scan.py audit` + `track`
**`audit` 命令**每次迭代跑一次,自动体检:
1. **覆盖缺口**: universe命中 vs 已建卡,找从未深挖的整个板块
2. **教训欠债**: 统计`[待自动化]`条数,提醒升级为代码
3. **过滤器松紧**: 若记分卡有数据,看否决股回报是否>买入股（过严的证据）
4. **缺口自问5连**: 固定反思清单（见下）

**`track` 命令**是学习引擎——记录每个决策的入场快照,日后对照实际回报打分:
- `track add`   : 为论点卡标的建入场快照（拉当前价,存scorecard.csv）
- `track grade` : 拉最新价,算自入场回报
- `track score` : 汇总命中率/胜率 → scorecard.md
**这是让框架"自证或自否"的关键**: 没有回报追踪,就永远不知道S1/S2b哪个真有效、过滤器是否过严。

## 每月/每财报季自反思例程（固定动作）
```bash
cd scanner
../.venv/bin/python scan.py scan            # 增量重扫(财报季后)
../.venv/bin/python scan.py edgar           # 补内生性数据
../.venv/bin/python scan.py report          # 带自动flag的新报告
../.venv/bin/python scan.py track --action grade   # 给已有决策打分
../.venv/bin/python scan.py audit           # 体检+缺口自问
```
然后人工回答 audit 输出的**缺口自问5连**:
1. 有无整个板块的信号命中从未深挖?（如医疗设备DXCM/COO/ALGN、公用事业）
2. 最近的否决理由,有无第3次重复却仍手动?→ 升级代码
3. 记分卡里否决股回报是否>买入股?→ 过滤器过严的证据
4. 有无新的失败模式本次才见到?→ 立即登记lessons.md
5. 数据源有无新的系统性盲区?（IFRS外国股/季度数不足/价格数据缺失）

## 已知的、机制主动暴露的当前缺口（2026-07-08 audit）
- **覆盖缺口**: 医疗设备（DXCM/COO/ALGN/BRKR,S1一整簇）、公用事业 从未深挖 → 潜在独立腿
- **教训欠债**: 8条`[待自动化]`未升级,最该先做L2(价格位置入报告,防VLO/STNG式接盘)
- **记分卡**: 已建14个入场快照,需时间跨度后grade,才能验证框架有效性
- **数据盲区**: 价格位置未进scan阶段(只在deep),导致"已涨到顶"要人工查;IFRS外国股拿不到us-gaap

## 元原则
**这个机制本身也要被它自己审查**——EVOLUTION.md 和 lessons.md 每季度复核,过时的规则删除,新失败模式补入。系统的进化能力,取决于这个闭环是否被诚实地执行。
