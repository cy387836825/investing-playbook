# 信号定义（Signal Definitions）参考

> ✅ 已统一(2026-07-08): 定义集中于 **signals.py**(唯一真源),backtest.py 和 scan.py 均从此导入。回测测的=实时flag的。

## 四个信号（回测版 backtest.py 的 SIGNALS）

### S2b — 营收加速
```
最新季营收同比 ≥ 25%  且  最新季同比 > 上一季同比(在加速)
```
- 同比: 该季营收 / 约1年前(350-380天)同季营收 - 1
- PIT: 只用 filed<=锚点 的季度数据

### S2a — 首次盈利
```
最新季 GAAP净利 > 0  且  前4季中 ≥3季 净利 ≤ 0
```
- 捕捉从亏损转盈利的拐点

### S1 — 周期反转
```
毛利率连续 ≥2季 环比改善  且  近4季TTM毛利率 < 历史基线
```
- 连续改善: 从最新季往前数,毛利率逐季递增的连续季数(一断即止)
- TTM = 近4季毛利率均值; 历史基线 = 第5季往前的毛利率均值
- 需 ≥8个季度毛利率数据
- 逻辑: 处于周期底部的均值回归早期

### S1超 — 超级周期/结构突破
```
毛利率连续 ≥2季 改善  且  TTM ≥ 历史基线  且  营收同比 ≥ 40%
```
- 与S1的区别: TTM已突破历史均值(天花板被抬高) + 营收强加速
- 逻辑: 需求范式改变的结构性突破(如MU的AI存储超级周期)
- 回测中表现最好: 季度采样18窗口有11个跑赢SPY,平均+17pp(但样本小,疑含AI牛市beta)

### 通用前置
- universe: 市值 ≥ $5B
- 数据源: SEC EDGAR companyfacts (PIT, 带filed日期)
- 营收标签: RevenueFromContractWithCustomer* / Revenues / SalesRevenueNet 等5个
- 净利: NetIncomeLoss ; 毛利: GrossProfit(部分公司无此标签→S1无法判定)
